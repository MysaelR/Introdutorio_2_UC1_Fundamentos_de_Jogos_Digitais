Software para console

Sistemas Operacionais

Antes de falarmos em específico sobre o software dedicado aos consoles, quero te apresentar uma introdução básica sobre o conceito de sistemas operacionais. Bem, basicamente um S.O. é um software que tem como função gerenciar os recursos do sistema. Ele define, por exemplo, quais os programas que devem ser processados pela CPU, o acesso a memória RAM, a criação e leitura de arquivos em discos, o controle dos barramentos da placa-mãe e todos os outros dispositivos de I/O (entrada e saída).

O SO é o encarregado de fazer a ponte entre software e hardware. Sendo assim, esses sistemas devem ser pensados para extrair o máximo de recursos do hardware e otimizar seu desempenho.

Independentemente da arquitetura de hardware (mobile, desktop, console), todo computador precisa de um SO para funcionar. Isso porque, ao ligarmos um dispositivo, as informações básicas são carregadas de uma memória não-volátil, conhecida como BIOS, que faz a checagem se os componentes estão em perfeito funcionamento. Após isso, a tarefa de controlar todo o processo é entregue ao SO, que geralmente está armazenado em disco. Ao término da inicialização do SO, o computador está pronto para ser utilizado. Por exemplo, quando desejamos instalar um novo jogo em nosso aparelho, é o Sistema Operacional quem vai interagir com o disco para verificar espaço e gravar todos os arquivos necessários. Quando instalado, ao iniciarmos o jogo, o SO carrega esses arquivos em disco para a memória RAM.

Atualmente há dois grandes grupos de SO disponíveis para os fabricantes de hardware, são eles: O Windows, fornecido pela Microsoft e o sistema UNIX. Esse último é um sistema básico, chamado de kernel, onde os desenvolvedores podem utilizá-los para criarem seus próprios sistemas operacionais. Por exemplo, todos os sistemas Linux, os sistemas da Apple e o próprio Android, da Google, são baseados em UNIX. Mas qual o impacto disso no desenvolvimento de jogos?

Para os desenvolvedores de games é necessário saber exatamente qual é o SO do dispositivo alvo para que o desenvolvimento possa ser direcionado a essa plataforma.

Em outras palavras, se eu pretendo desenvolver um game para Windows, então é necessário trabalhar com as bibliotecas próprias para Windows. Os desktops e mobiles, por serem computadores de uso geral, possuem seus SO`s mais acessíveis e é mais fácil desenvolver games para essas plataformas.

Em contrapartida, os consoles são de uso restrito (apenas para jogos e entretenimento) e possuem SO`s mais específicos. Sendo assim, o processo de desenvolvimento para console requer equipes mais qualificadas e conhecimento profundo sobre o SO da plataforma.

Vamos então conhecer características dos principais sistemas operacionais de games? Você não pode perder esse conhecimento.

Sistema operacional Playstation

A família PS, desde a versão 4, é equipada com um Sistema Operacional chamado Orbis, que por sua vez é baseado no sistema Linux FreeBSD. A grande diferença para os sistemas que rodam em desktop (como o FreeBSD) é que o Orbis fornece API`s específicas para games.

Uma API (interfação de programação de aplicativos) é um conjunto de rotinas que os softwares podem disponibilizar aos desenvolvedores. Por exemplo, o Maps é uma API do Google utilizada para ter acesso ao sistema de mapas e coordenadas. A vantagem dessas interfaces é que o programador não precisa saber como uma API funciona, mas sim quais as informações que devem ser enviadas a ela.

No caso do Orbis, há API`s específicas para o gerenciamento de GPUs e também para dispositivos de realidade virtual, tal como óculos. Assim, os desenvolvedores de games para PS podem gerenciar o processamento gráfico de seus games de forma muito mais eficiente, devido às características do sistema operacional Orbis.

Sistema operacional Xbox

O mesmo conceito utilizado para Playstation vale para Xbox. como é um sistema dedicado a jogos e entretenimento, o SO da plataforma Xbox também é específico. Neste caso, ele é baseado em Windows e se chama OneCore. Assim como em Playstation, a diferença do OneCore para um Windows tradicional é que ele disponibiliza APIs específicas para o processamento de imagens.

Como você já pode ter imaginado, por se tratar de sistemas operacionais distintos, os jogos feitos para Playstation e XBOX não são compatíveis. Os estúdios que desejam desenvolver para ambas as plataformas devem ter desenvolvimento específicos para cada uma delas.

Componentes de um sistema operacional

A definição de sistemas operacionais se divide em duas partes: a primeira diz que o SO é um intermediário, não sendo nem software nem hardware, mas sim uma ponte entre os dois elementos.

A segunda, que o SO é um gerenciador de recursos que controla as aplicações a serem executadas e todos os recursos de hardware que elas consomem. Aqui, essa segunda visão se encaixa melhor e nos possibilita entender como é feito esse gerenciamento.

Vamos conhecer como funciona cada um deles e qual o seu papel nesse sistema de controle:

Gerenciamento de processos

Para rodar um software é necessário vários processos sendo executados ao mesmo tempo. Para melhor compreensão, dê o play no vídeo a seguir.

Gerenciamento de memória

A função principal desse módulo do SO é gerenciar o funcionamento da memória RAM. Como sabemos, o acesso a dados presentes em disco é bem lento. Assim, a estratégia é carregar esses dados diretamente na memória RAM, para que o acesso seja mais rápido e consequentemente seu software tenha um desempenho satisfatório. Cada célula da RAM possui um endereço, e são nesses endereços que são armazenados os dados. Para melhor compreensão, dê o play no vídeo a seguir.

Gerenciamento de recursos

Esse é um módulo de extrema importância. A CPU é o principal componente do computador e todos os softwares precisam utilizá-la. Como então gerenciar as solicitações conflitantes, ou seja, aquelas que chegam ao mesmo tempo à CPU?

O módulo de gerenciamento de recursos é quem organiza essa fila. E não somente da CPU, mas de todos os recursos críticos: GPU, barramentos (que são as “estradas” por onde trafegam os dados), RAM e armazenamento em disco.

Assim, o SO fornece prioridades aos processos (isso funciona de forma análoga a um hospital: não importa quanto tempo você está esperando, se chegar um paciente com um caso mais grave, os recursos médicos serão destinados a ele) e organiza quem deve ser atendido primeiro. Por exemplo, softwares anti-vírus ou mecanismos de proteção a falhas e erros (como por exemplo, divisão por zero), possuem as mais altas prioridades dentro do SO.

Entrada e saída de dados (I/O)

Os dados que são processados pelo computador são, na maioria das vezes, oriundos de dispositivos externos (mouse, teclado, joystick etc) e são exibidos aos usuários também por dispositivos externos (tela, monitor, impressora). Esses dispositivos são compartilhados por todos os processos (afinal, você só tem um teclado em seu PC, tanto para digitar um texto no Word quanto para um endereço no browser).

Você já pode imaginar que há uma forte concorrência entre os softwares para o acesso a esses dispositivos. Logo, a função principal desse módulo de I/O do sistema operacional é controlar essas “disputas”. Além disso, o I/O também gerencia como as informações chegam e partem da CPU com destino a esses dispositivos. Por exemplo, existem inúmeros fabricantes de mouses, cada um com sua configuração.

A CPU não conhece todos esses fabricantes e como todos os mouses funcionam. Ela simplesmente recebe, processa e envia os dados. A função de conhecer todos os mouses (e todos os demais dispositivos periféricos) é do SO, mais precisamente do módulo I/O.

Portanto, os fabricantes de periféricos devem fornecer aos desenvolvedores de SO um pequeno software que mostra ao módulo I/O como ele deve converter o sinal enviado pelo periférico em informações legíveis para a CPU. Esses pequenos softwares são chamados de drivers. A maioria dos periféricos são plug-and-play, ou seja, seus drivers já estão embutidos no SO. Outros, como impressoras ou óculos VR, é necessário baixar o software de instalação em um site na internet.

Sistema de arquivos

Como sabemos, a memória RAM é volátil, isto é, ao deixar de ser alimentada com corrente elétrica, os dados presentes são apagados. Isso leva a necessidade de se gravar informações em sistemas não-voláteis (como discos e cartões de memória). Imagina você não poder gravar o progresso de seu jogo, e ao desligar o videogame perder tudo o que você fez.

Para que isso não aconteça, o SO possui um módulo de gerencia de arquivos, que faz a leitura e gravação de dados em disco. Assim, esse módulo tanto carrega os dados em disco para a memória quanto “salva” as informações da RAM em disco.

Para aumentar o desempenho e prolongar a vida útil do disco, o SO não apaga os dados diretamente no disco (ao menos que você o formate). Quando você apaga um arquivo no disco, na verdade ele continua fisicamente alí e a estratégia “esperta” do SO é disponibilizar a área ocupada pelo arquivo para uma sobrescrita. Isso poupa a vida útil do disco (cada processo de formatação diminui o tempo de duração dos sistemas semicondutores) e aumenta o desempenho (apagar literalmente um dado do disco demora bastante tempo).

Sistemas multiprocessadores

Atualmente a maioria dos processadores possuem diversos núcleos de processamento, conhecidos como Core. Eles se comportam como unidades de processamento independentes e cabe ao SO direcionar para qual núcleo a informação será processada.

Sendo assim, os sistemas operacionais devem ser capazes de lidar com esse tipo de configuração de processamento. Para tanto, eles podem ser de dois tipos: simétricos, quando todos os núcleos da CPU possuem a mesma função; ou assimétricos, que funciona na configuração master-slave: um núcleo principal (master), geralmente dedicado às tarefas do SO, e os demais (slaves) subordinados ao master.

Por fim, temos os sistemas operacionais distribuídos. Eles estão presentes em clusters de computadores e são responsáveis pelo gerenciamento de recursos e processos em seus próprios dispositivos quanto enviar mensagens para os demais computadores do cluster. Esse tipo de configuração é utilizado em sistemas que demandam alta capacidade de processamento, tais como servidores. Assim, aquele seu jogo on-line favorito muito provavelmente está rodando em um cluster computacional.

Um cluster computacional é um sistema composto por diversos computadores, cada um com seu SO, CPU e memória próprios, que atuam como se fossem um único computador. Os dados a processar são divididos entre os diversos computadores, que trocam informações entre si através de sistemas de rede (cabo ou wireless).
